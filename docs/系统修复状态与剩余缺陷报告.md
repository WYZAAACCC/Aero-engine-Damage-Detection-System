## 二、底层算法/模型不完善清单

### 2.1 视觉检测算法

| 资产 | 当前实现 | 真实论文方法 | 差距 |
|------|---------|-------------|------|
| **CA²** | ResNet-50 特征提取 (ImageNet预训练) | MMR 密度估计 + 正常特征库 KNN | 缺正常样本特征库、缺 KNN 模型、缺校准阈值、缺 OOD 检测 |
| **SLF-YOLO** | Sobel 梯度边缘检测 (基线) | YOLOv8 + 特征融合 + NEU-DET/GC10-DET 训练权重 | **完全未实现**。需要训练数据 + 训练代码 + 领域权重 |
| **SAM-Adapter** | SAM ViT-H + 中心点提示 / Canny 边缘 | SAM + Crack Adapter/LoRA + 检测器 prompt | 缺裂纹 Adapter 权重、缺上游检测器 prompt 点、缺细长形态后处理 |
| **EGCIENet** | 图像统计分析 | SegFormer + SAM 边缘引导 | **完全未实现**。论文代码/权重未公开 |
| **TS-SAM** | 不可用 | SAM ViT-H + CSA+MRM+FFD 双流 | **完全未实现**。需 2.4GB ViT-H + adapter 权重 |

### 2.2 信号检测算法

| 资产 | 当前实现 | 真实论文方法 | 差距 |
|------|---------|-------------|------|
| **WCamba** | Hilbert 包络谱 + 固定倍频 | CNN+Mamba + CWRU/PU 训练权重 | **完全未实现**。需训练数据 + 训练代码 + 模型权重 + 轴承几何参数（BPFO/BPFI/BSF/FTF） |
| **FaultSense** | 滑动窗口线性趋势 | TensorFlow LSTM-Autoencoder + C-MAPSS 训练 | **完全未实现**。需 C-MAPSS 训练 + .h5 权重 + 工况归一化 |
| **Isolation Forest** | sklearn IsolationForest | 同上但不能直接用 | 缺正常基线模型、缺工况归一化、缺校准阈值 |

### 2.3 表征算法

| 资产 | 当前实现 | 工程要求 | 差距 |
|------|---------|---------|------|
| **裂纹几何测量** | bbox 矩形估计 | 骨架化长度 + mask 面积 + 分支计数 + 曲率 + 方向 | 缺 skeletonize、缺主路径长度、缺最大/平均宽度、缺曲率、缺端点检测 |
| **损伤类型分类** | 关键词匹配（11 类×10 关键词） | 训练分类器或至少校准的关键词模型 | 缺 score_semantics 处理、缺概率校准、缺混淆矩阵 |
| **严重度分级** | 通用阈值 (0.5/2/5mm) | 发动机型号特定、部件位置特定、AMM/ESM 绑定 | 缺机型绑定、缺位置特定、缺材料特定、缺可测量性/误差 |

### 2.4 可靠性/寿命模型

| 资产 | 当前实现 | 真实方法 | 差距 |
|------|---------|---------|------|
| **Paris Law** | 数值积分 (da/dN=C·ΔK^m) | 物理方程 (py_fatigue/NASGRO) | 缺 ΔKth/Kc 检查、缺短裂纹/高温蠕变 LEFM 适用域、缺载荷谱、缺材料数据库 |
| **CNN-LSTM RUL** | 线性趋势外推 | CNN-LSTM + C-MAPSS FD001-004 训练 | **完全未实现**。资产 ID 与实际实现不匹配 |
| **FDPP 概率裂纹** | Monte Carlo + Paris (≤500 样本) | 完整 FDPP 框架 | 缺相关性矩阵、缺重要性采样、缺 FORM/SORM |
| **pyLife S-N** | Basquin 公式 | pyLife FKM 非线性 | 缺 pyLife 库、缺 FKM 非线性修正 |
| **PINN 机队预测** | 返回 unavailable | PINN + Paris + 机队载荷谱 | **完全未实现** |

## 六、关键结论

1. **v0.1.1 修复的本质**：让系统**停止撒谎**。所有算法现在诚实报告自己是否可用、结果是否可信、能否影响安全决策。0 个检测器/可靠性模型标记为 "valid"。

2. **Agent 闭环是最大缺口**：AER-001 (/runs 未实现) 和 AER-002 (Agent 未接入) 是当前最阻碍系统闭环的两个问题。没有它们，系统只能手工调用资产而不能让 Agent 自主完成完整诊断。

3. **算法层面**：31 个资产中只有 3 个是真正工程有效的。其余都是基线替代、占位符或需训练权重的 DL 模型。补齐 DL 模型权重是使资产从 "名义存在" 变为 "工程有效" 的核心工作。

4. **知识系统**：29 条手写知识与文档 RAG 之间有巨大的工程实现差距。这是确保系统不会被错误知识误导的关键。

5. **安全关键交付**：当前 0/31 资产可安全用于真实维修决策。系统从"研究原型"到"受控试验"需要完成 Phase 1-3 的全部工作，到"安全关键可用"还需要 Phase 4-6 + 适航和维修体系批准。
