## Phase 1: 六条真实闭环（最优先）

### 1. CA² / MMR — 叶片异常检测

| 项目 | 状态 |
|------|------|
| 指导书要求 | MMR 或 PatchCore + AeBAD 数据集 |
| 当前状态 | ⚠️ **替代训练完成** (PatchCore, 非 MMR) |
| 训练数据 | AeBAD-S：521 train / 1818 test |
| 训练结果 | AUROC=0.605, F1=0.633 |
| 产物路径 | `/f/aero-training/artifacts/models/patchcore_aebad/v1.0/model.pth` |

**为什么没用 MMR？**
MMR 需要 MAE ViT-base 预训练权重 + 对抗训练，显存需求 16-24GB，8GB 不够。PatchCore 使用 WideResNet-50 + coreset memory bank，8GB 可运行，功能等价。

**差距：**
- PatchCore vs MMR：MMR 在 AeBAD 上 AUROC 更高（~0.75+），因为有掩码重建机制应对域偏移
- AeBAD 本身的域偏移（光照/视角变化）使所有方法都难以达到 >0.8 AUROC

### 2. EGCIENet — 叶片缺陷分割

| 项目 | 状态 |
|------|------|
| 指导书要求 | SegFormer MiT-B3 + 边缘引导 + AEBIS |
| 当前状态 | ⚠️ **替代训练完成** (UNet+ResNet18, 非 SegFormer) |
| 训练数据 | AEBIS：422 train / 59 test |
| 训练结果 | Dice=0.787 |
| 产物路径 | `/f/aero-training/artifacts/models/unet_aebis/v1.0/best_model.pth` |

**为什么没用 SegFormer？**
1. SegFormer MiT-B3 需要 12-16GB VRAM（8GB 不够）
2. EGCIENet 上游代码存在数据加载 bug（指导书 Section 5.1）
3. UNet+ResNet18（17.6M 参数）可在 8GB VRAM 上以 batch=8 训练

**差距：**
- UNet 无边缘引导分支（EGCIENet 的核心创新）
- UNet 的边界精度可能低于 SegFormer
- 功能上：同样是二值叶片缺陷分割，同样的 AEBIS 数据

## Phase 2: 扩展模型

### 7. SAM-Adapter — 裂纹分割

| 项目 | 状态 |
|------|------|
| 当前状态 | ❌ **未训练** |
| 阻塞原因 | **显存不足** |

**详细分析：**
- SAM ViT-B 基础模型需要 ~4GB VRAM（仅加载）
- 加上 Adapter/LoRA 训练 + 梯度 + batch → 需要 12-16GB
- 当前 GPU：NVIDIA RTX 4060 Laptop, 8GB VRAM
- 可用的 LoRA-SAM 裂纹权重存在（[HuggingFace](https://huggingface.co/zxy239/LoRA-SAM-for-damage-detection)），但是针对砖石裂纹，非航空叶片

**需要什么才能训练：**
- ≥12GB VRAM GPU（如 RTX 4070/4080 桌面版，或云 GPU A10/A100）
- 或使用更轻的 EdgeSAM（ConvLoRA 变体，可在 8GB 上运行）
- 航空发动机叶片裂纹标注数据（AeBAD 有裂纹掩膜但需转换）

### 8. TS-SAM — 双流叶片分割

| 项目 | 状态 |
|------|------|
| 当前状态 | ❌ **未训练** |
| 阻塞原因 | **显存严重不足** |

**详细分析：**
- TS-SAM 需要 SAM ViT-H 基础模型（~2.4GB 权重文件）
- 加上双流 CSA+MRM+FFD 模块 → 推理需要 16GB+，训练需要 24GB+
- 8GB VRAM 完全不够
- 官方代码使用 Python 3.8 + PyTorch 1.13 + MMCV 1.7，需单独容器

**需要什么才能训练：**
- ≥24GB VRAM GPU（RTX 4090、A5000、A100 等）
- 单独的旧版 Python 环境容器
- AEBIS 数据已就绪

| 项目 | 状态 |
|------|------|
| 当前状态 | ❌ **未训练 — 数据未下载** |

**阻塞原因：**
1. Paderborn 数据集需要从大学官网申请下载（需要填写使用协议）

**需要什么：**
- 访问 https://mb.uni-paderborn.de/kat/forschung/bearing-datacenter 申请数据
- 下载后可用 WCamba 训练脚本改编

### 11. Isolation Forest — 多传感器异常检测

| 项目 | 状态 |
|------|------|
| 当前状态 | ❌ **未训练 — 缺少真实发动机遥测数据** |

**阻塞原因：**
- 系统要求 7 个具名传感器（排气温度、振动、轴承温度、入口压力、润滑油压力、燃油流量）
- **公开 C-MAPSS 传感器是匿名的（s1-s21）**，不能诚实地映射为物理传感器名
- 使用 C-MAPSS 训练的研究代理模型不能标记为生产就绪

**需要什么：**
- 真实发动机遥测数据（Parquet 格式，含 engine_id/timestamp/operating_regime）
- 专家确认的健康期标签
- 已知维修事件用于校准

| 项目 | 状态 |
|------|------|
| 当前状态 | ⏸️ **延后** |

**阻塞原因（指导书 Section 13.3）：**
- 需要物理方程、状态变量、边界条件——由领域工程师提供

**代码 Agent 无法自动完成。**
